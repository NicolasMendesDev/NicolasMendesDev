## ⚡ About Me

I'm not just a developer — I'm an **engineer with a security instinct**.

My work lives at the intersection of **software craftsmanship**, **cybersecurity research**, and **systems thinking**. I don't ship code that simply *works* — I ship code that works *correctly, securely, and at scale*.

Every application I build starts with a threat model. Every system I architect assumes it will be attacked. This isn't paranoia — it's engineering discipline.

**What drives me:**

- 🏗️ Architecting systems that outlast trends and resist entropy
- 🔐 Securing what others overlook — before attackers discover it
- 🤖 Leveraging AI as a force multiplier for engineering productivity
- 🌐 Contributing to open source because compounding knowledge is power
- 📈 Treating continuous learning as a competitive advantage, not a chore

I'm building toward becoming a **tech founder who understands both the system's blueprints and its fault lines** — because the best CTOs think like the attackers trying to break what they built.

> *The gap between a developer and an engineer is the same as between someone who writes code and someone who designs systems. I'm interested in the latter.*

## ⚙️ Engineering Principles

```
  ⟨01⟩  SECURITY FIRST        →  Every line of code is an attack surface — design accordingly
  ⟨02⟩  CLEAN ARCHITECTURE    →  Build systems that outlast their creators and resist entropy
  ⟨03⟩  AUTOMATION BY DEFAULT →  If you do it twice, automate it. If it breaks, fix the root
  ⟨04⟩  PERFORMANCE MATTERS   →  Microseconds compound into seconds at the scale that counts
  ⟨05⟩  FAIL LOUDLY           →  Silent failures are more dangerous than visible crashes
  ⟨06⟩  DOCUMENT THE WHY      →  Code explains what. Comments explain why. Docs save teams
  ⟨07⟩  SHIP & ITERATE        →  Perfect is the enemy of deployed — but deployed must be secure
  ⟨08⟩  LEARN IN PUBLIC       →  The fastest way to master something is to teach it
```

## 🌍 Open Source Philosophy

<div align="center">

> *"Open source is not charity — it's compound interest on collective intelligence.*
> *Every contribution multiplies across thousands of projects you'll never know you shaped."*

</div>

<br/>

I believe in the power of **building in public**. Open source is how we:

- 🤝 **Collaborate** — great software is rarely built alone; the best ideas come from friction between perspectives
- 📖 **Learn** — reading real production codebases teaches what tutorials deliberately hide
- 🔍 **Audit** — security fundamentally improves when code is visible, reviewable, and verifiable
- 🌱 **Grow** — today's contributors are tomorrow's architects, and the ecosystem compounds

I actively contribute, open issues, write documentation, and engage with maintainers. Every pull request is a letter to engineers you've never met.
